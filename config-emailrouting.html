How to Set Up Email Routing in Genesys Cloud

Overview

This guide walks you through setting up an email channel in Genesys Cloud for routing incoming messages to specific queues or flows. You can use Genesys-provided domains or configure a custom domain with your SMTP/Graph API setup.

Step 1: Access Email Configuration

Go to Contact Center > Email in the Genesys Cloud admin interface.

Step 2: Choose or Set Up Your Domain

  • Option 1: Use Genesys Cloud (mypurecloud.io) — Quick and easy for testing or small teams.
  • Option 2: Use a Custom Domain
    • You’ll need to verify domain ownership via DNS or email validation.
    • Choose either:
      • SMTP — Connects to your mail server.
      • Microsoft Graph API — Recommended for Outlook/Exchange environments.

Step 3: Configure Email Address

Once the domain is active, set up your routing address:

  • From Name: The display name seen by users
  • Email Address: The address that will receive routed messages
  • Reply To: Optional override for reply address

Step 4: Set Routing Rules

  • Always route to this flow or queue: Enable this option
  • Select a Flow or Queue: Choose one you’ve prebuilt for email handling
  • Optional Routing Enhancements:
    • Assign Skills to route based on agent expertise
    • Select preferred Language for agent matching
    • Set Priority to define email importance level

Step 5: Enable Additional Options

  • Email Signature: Toggle on if you want to append a signature to replies

Step 6: Testing

  • Send a test email to the configured address
  • Verify that it reaches the correct queue and appears in the agent interface
  • Confirm the signature, routing, and reply behavior all function as expected
