Web Messenger – Exercise 4

Add Digital Bot Flow & Configure Copilot with Data Table Lookup

Objective

Enhance your Architect flow by integrating Digital Bot for inbound messages, use data tables for routing, and configure Copilot with relevant articles.

Steps

  1. Integrate Digital Bot Flow
    • Create or import a Digital Bot Flow for handling common chat inquiries
    • In your Architect inbound chat flow, add a step to invoke the Digital Bot flow
    • Use a Data Table Lookup to route chats based on business unit as in previous exercise
  2. Set Up Data Table for Routing & Articles
    • Expand your existing data table to include at least 10 relevant article references for Copilot
    • Articles should cover FAQs, troubleshooting, and common workflows
    • Include columns for BU, Article Titles, Article Links, and Keywords
  3. Configure Copilot for Queues
    • Assign Copilot to each queue in Genesys Cloud
    • Link Copilot with your Knowledge Base containing the 10+ articles
    • Enable Answer Generation and Highlights features
    • Test Copilot response quality on different chat scenarios
  4. Test & Validate
    • Start inbound chats on the web messenger
    • Verify Digital Bot handles initial queries correctly
    • Confirm routing to appropriate queues via data table lookup
    • Check Copilot delivers helpful suggestions and article links
