Web Messenger – Exercise 1

Create a Simple Web Messenger Configuration in Genesys Cloud

Objective

Set up a basic web messenger deployment with direct agent routing using Architect.

Steps

  1. Configure the Messenger
    • Go to Admin > Messenger Configuration
    • Create a new configuration or edit an existing one
    • Set appearance: colors, logo, language, and avatar
    • Enable Auto-Start to launch the messenger automatically
  2. Create a Simple Queue
    • Go to Admin > Queues
    • Create a new queue with a recognizable name like SimpleWebChatQueue
    • Assign agents or yourself for testing
  3. Set Up Deployment
    • Go to Messenger Deployment
    • Select the Messenger config and assign the queue via an Architect flow
    • Whitelist your website domain
    • Copy and embed the script tag into your website
  4. Create an Architect Flow
    • Go to Architect > Digital Bot Flows
    • Create a new Inbound Message Flow
    • Add a Transfer to ACD action and route it to SimpleWebChatQueue
    • Publish the flow and connect it to the deployment

Test It Out

  • Open the website where you installed the widget
  • Initiate a chat and verify it routes directly to the assigned queue
  • Make sure an agent can accept and respond to the message
