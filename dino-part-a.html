Part A: Dino Park Setup

This section covers the foundational configuration tasks necessary to prepare the Dino Park environment. Here you will set up key Genesys Cloud components and define your initial interaction architecture.

Setup Objectives

  • Provision essential components such as Virtual Assistants, Queues, and Users
  • Define basic routing and escalation rules for incoming customer requests
  • Establish baseline monitoring and reporting dashboards
  • Prepare the environment for integration with digital engagement tools

Setup Steps

  1. Create Virtual Assistants and train intents for common queries
  2. Configure queues and assign agents with appropriate skills
  3. Establish escalation rules to handoff interactions from bots to agents
  4. Set up dashboards and analytics reports to monitor performance
  5. Test initial voice and chat interaction flows

Best Practices

  • Document each configuration step for team collaboration
  • Use naming conventions to keep assets organized and discoverable
  • Test early and often to identify configuration gaps
  • Leverage Genesys documentation and community forums for support

Ready to Move On?

After completing these setup tasks, you're ready to design the customer interaction flows in Part B Digital Flow. This will bring your Dino Park support experience to life!
